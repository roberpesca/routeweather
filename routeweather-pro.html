OAuth Flow — Step by Step
1
Register your app on Strava
Go to strava.com/settings/api → create app → note your client_id and client_secret. Set your callback URL to this page's URL.
2
Redirect user to Strava login
This app sends you to Strava's authorization page with your client_id, requested scope, and redirect URI. Scope needed: read,activity:read_all.
3
Exchange auth code for token
Strava redirects back with a one-time code. The app POSTs to Strava's token endpoint with the code + your client_secret to receive the access_token. Note: for production apps, use a backend server to protect your client_secret.
4
Fetch routes from Strava
The app calls GET /api/v3/athlete/routes to list your saved routes. Click one to load its polyline and analyze weather along the route.
5
Decode polyline & fetch weather
The route's summary_polyline is decoded using Google's polyline algorithm into lat/lng points. These feed into the same weather analysis pipeline as GPX files.
Key Code Snippets
1. Frontend — Redirect to Strava OAuth
const authUrl = new URL('https://www.strava.com/oauth/authorize'); authUrl.searchParams.set('client_id', CLIENT_ID); authUrl.searchParams.set('redirect_uri', 'https://yourapp.com/strava/callback'); authUrl.searchParams.set('response_type','code'); authUrl.searchParams.set('scope', 'read,activity:read_all'); window.location.href = authUrl.toString();
2. Backend — Exchange code for token (Node/Express)
app.get('/strava/callback', async (req, res) => { const { code } = req.query; const tokenRes = await fetch('https://www.strava.com/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ client_id, client_secret, code, grant_type: 'authorization_code' }) }); const { access_token, refresh_token, expires_at, athlete } = await tokenRes.json(); // Save to DB, return session JWT to frontend await db.saveToken(athlete.id, { access_token, refresh_token, expires_at }); res.redirect(`/app?session=${issueJWT(athlete.id)}`); });
3. Backend — Fetch route GPS stream
async function getActivityGPS(activityId, accessToken) { const streams = await fetch( `https://www.strava.com/api/v3/activities/${activityId}/streams` + '?keys=latlng,altitude&key_by_type=true', { headers: { Authorization: `Bearer ${accessToken}` } } ).then(r => r.json()); // Convert to same format as GPX parser output return streams.latlng.data.map(([lat, lon], i) => ({ lat, lon, ele: streams.altitude?.data[i] ?? 0 })); }
4. Token refresh (run before every Strava API call)
async function freshToken(userId) { const token = await db.getToken(userId); if (token.expires_at * 1000 > Date.now()) return token.access_token; const refreshed = await fetch('https://www.strava.com/oauth/token', { method: 'POST', body: JSON.stringify({ client_id, client_secret, grant_type: 'refresh_token', refresh_token: token.refresh_token }) }).then(r => r.json()); await db.saveToken(userId, refreshed); return refreshed.access_token; }
What You Need to Build
🔑
Strava API credentials
Register at strava.com/settings/api. Free tier allows 100 requests/15 min, 1000/day — plenty for a weather app.
🖥
Small backend server (for production)
For production, use a backend to keep your client_secret safe. This app does client-side OAuth for convenience — suitable for personal use.
🗄
Token storage
This app stores the token in memory for the current session only. For production, use a database (Postgres / Redis).
☁️
Weather API — already free
Open-Meteo requires no API key, handles forecast + historical ERA5 data. For enterprise accuracy, Tomorrow.io or Meteomatics offer hyperlocal forecasts.
🔔
Push notifications (optional)
For reliable cross-device notifications: register a Service Worker + use the Web Push API (VAPID keys). Or use email via Resend/SendGrid for pre-ride alerts.
